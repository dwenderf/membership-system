## Component Reuse Summary

**Existing Components to Reuse:**
1. **AlternatesManager** - Used as-is in `/user/captain/[id]/alternates`
2. **Registration tiles** - Same style as `/admin/reports/registrations`
3. **Table components** - Similar to admin roster tables (sortable, searchable)
4. **User search/autocomplete** - For adding captains in admin

**New Components Needed:**
1. **CaptainDashboard** - Registration tiles for captain view
2. **CaptainRosterTable** - Roster table with payment status (no edit/actions)
3. **CaptainTab** - Navigation tab (conditional rendering)
4. **CaptainAssignmentSection** - Admin UI for managing captains
5. **CaptainTileInfo** - Captain display on registration tiles
6. **CaptainNotificationSettings** - Settings page (Phase 3)

## Responsive Design Notes

**Mobile Considerations:**
- Tables should scroll horizontally on mobile
- Tiles stack vertically (already responsive)
- Search inputs full-width on mobile
- Action buttons stack vertically on small screens
- Tab navigation may collapse to dropdown on mobile

**Tablet:**
- 2-column tile layout for captain dashboard
- Tables still scrollable if needed
- Full navigation visible

**Desktop:**
- 3-column tile layout for captain dashboard (if space)
- Full tables visible
- All navigation and actions inline

---

## Accessibility Notes

- All interactive elements keyboard accessible
- ARIA labels for screen readers
- Focus indicators on all clickable elements
- Color not sole indicator (use icons + text for payment status)
- Skip links for navigation
- Semantic HTML (tables for data, proper headings)
